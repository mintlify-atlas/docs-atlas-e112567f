The Courser API is a REST API that lets you build and interact with AI-powered course chatbots. All requests and responses use JSON.

## Base URL

<CodeGroup>

```bash Local development
http://localhost:8000
```

```bash Production
https://your-deployed-url.com
```

</CodeGroup>

<Info>
Replace `https://your-deployed-url.com` with the URL of your deployed Courser instance.
</Info>

## Authentication

The Courser API uses JWT (JSON Web Token) bearer authentication. Once you obtain a token via any auth endpoint, pass it with every authenticated request using the header below.

```
x-access'courser-auth-token: <your_jwt_token>
```

<Warning>
The header name contains an apostrophe: `x-access'courser-auth-token`. This is intentional and must be sent exactly as shown.
</Warning>

Tokens are valid for **1000 days** from the time of issue. See the [Authentication](/api/authentication) page for full details.

## Endpoints

### Auth ‚Äî `/auth/*`

| Method | Path | Auth required | Description |
|--------|------|--------------|-------------|
| `POST` | `/auth/google` | No | Sign in or register with Google OAuth |
| `POST` | `/auth/signup-email` | No | Register a new account with email and password |
| `POST` | `/auth/login-email` | No | Sign in with email and password |
| `GET` | `/auth/isloggedin` | Yes | Verify a token and return the authenticated user's profile |

### Chatbot ‚Äî `/chatbot/*`

| Method | Path | Auth required | Description |
|--------|------|--------------|-------------|
| `POST` | `/chatbot/chat` | No | Send a question to a course chatbot and get a cited answer |
| `POST` | `/chatbot/create` | Yes | Create a new chatbot from YouTube lecture URLs |
| `GET` | `/chatbot/:courseId/dataset/count` | Yes | Get the total conversation count for a course |
| `GET` | `/chatbot/:courseId/dataset/export` | Yes | Download recent conversations as a CSV file |

### Course ‚Äî `/course/*`

| Method | Path | Auth required | Description |
|--------|------|--------------|-------------|
| `GET` | `/course/getAllCourses` | Yes | Get all courses belonging to the authenticated user |
| `GET` | `/course/getCourse/:courseID` | No | Get a course by ID (public ‚Äî used by the chatbot widget) |
| `POST` | `/course/update` | Yes | Update a course's settings, including image upload |
| `GET` | `/course/getCourseProtected/:courseID` | Yes | Get a course by ID (authenticated version) |

## Error responses

| Status | Meaning |
|--------|---------|
| `401` | Unauthorized ‚Äî missing, invalid, or expired token, or no matching user |
| `409` | Conflict ‚Äî a resource with the same identifier (e.g. email address) already exists |
| `500` | Internal server error ‚Äî an unexpected error occurred on the server |
